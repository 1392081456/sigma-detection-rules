# Hunt — GeoServer XPath Expression Injection (CVE-2024-36401)

> Look for OGC request parameters (`propertyName`, `valueReference`, `expression`) containing Java method invocation syntax. The bug evaluates these parameters as XPath expressions via the GeoTools `commons-jxpath` library, which allows extension methods that call arbitrary Java.

## Detection key

GeoServer accepts OGC WFS / WMS / WPS requests where `propertyName` is supposed to be a simple field name like `the_geom`. The vulnerable code passes it to a `jxpath` evaluator that supports extension methods using the `exec(...)` or `Runtime.getRuntime().exec(...)` syntax.

Indicator: `Runtime`, `exec(`, `ProcessBuilder`, or `getRuntime` appearing in `propertyName` / `valueReference` / `expression` query parameters of WFS/WMS/WPS endpoints.

## Splunk SPL

```spl
index=webserver earliest=-90d uri_path="*/geoserver/*"
| where match(uri_query, "(?i)(propertyName|valueReference|expression)=[^&]*(Runtime|getRuntime|exec\(|ProcessBuilder)")
| stats count, values(uri_path) as paths, values(http_method) as methods by src_ip
| sort -count
```

## Microsoft Sentinel KQL

```kusto
let lookback = 90d;
W3CIISLog
| where TimeGenerated > ago(lookback)
| where csUriStem contains "/geoserver/"
| where csUriQuery matches regex @"(?i)(propertyName|valueReference|expression)=[^&]*(Runtime|getRuntime|exec\(|ProcessBuilder)"
| summarize hits = count(), paths = make_set(csUriStem, 5), methods = make_set(csMethod, 5) by cIP
| order by hits desc
```

## Elastic ES|QL

```esql
FROM logs-apache.access-*
| WHERE @timestamp > NOW() - 90 days
| WHERE url.path LIKE "*/geoserver/*"
| WHERE url.query RLIKE "(?i)(propertyName|valueReference|expression)=[^&]*(Runtime|getRuntime|exec\\(|ProcessBuilder)"
| STATS hits = COUNT(*), paths = VALUES(url.path) BY source.ip
| SORT hits DESC
```

## Post-hunt pivot — confirm command execution

GeoServer runs as the `geoserver` system user (or root in some Docker setups). A successful exploitation results in process creation by the GeoServer JVM. Hunt for child processes of the GeoServer process:

```spl
index=hosts earliest=-90d
| where parent_process_cmdline="*geoserver*" OR parent_process_name="java"
| where process_name IN ("sh", "bash", "cmd.exe", "curl", "wget", "nc", "id", "whoami", "uname")
| stats earliest(_time) as first_seen, latest(_time) as last_seen, count by host, process_name, process_cmdline
| sort -count
```

The pairing "OGC request with `Runtime` in `propertyName` at time T" + "geoserver JVM spawned shell at time T+1 to T+5" confirms successful exploitation. The shell command in the second event is what was executed.
