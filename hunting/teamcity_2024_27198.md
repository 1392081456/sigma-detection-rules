# Hunt — TeamCity Authentication Bypass (CVE-2024-27198)

> Look for **URL paths containing `;` immediately before `/admin/`** or any REST API endpoint. The TeamCity bug treats the path-parameter separator as a hint to skip authentication on selected REST routes.

## Detection key

The TeamCity servlet's path parsing splits on `;`, but its authentication filter looks at the **un-split** request URI for matching. So a request to `/app/rest/users;foo=bar/` matches the auth filter as "non-protected static path" (because of `;foo=bar`) but reaches the REST handler as `/app/rest/users` (authenticated endpoint). The result is unauthenticated REST API access — including admin account creation.

Indicator: `;` in the URL path anywhere **before** `/admin/` or `/app/rest/`.

## Splunk SPL

```spl
index=webserver earliest=-90d
| where match(uri_path, ";[^/]+/(?:admin|app/rest)")
| stats count, dc(http_method) as methods, values(uri_path) as paths by src_ip
| sort -count
```

## Microsoft Sentinel KQL

```kusto
let lookback = 90d;
W3CIISLog
| where TimeGenerated > ago(lookback)
| where csUriStem matches regex @";[^/]+/(?:admin|app/rest)"
| summarize hits = count(), methods = make_set(csMethod, 5), paths = make_set(csUriStem, 5) by cIP
| order by hits desc
```

## Elastic ES|QL

```esql
FROM logs-apache.access-*
| WHERE @timestamp > NOW() - 90 days
| WHERE url.path RLIKE ";[^/]+/(admin|app/rest)"
| STATS hits = COUNT(*), methods = VALUES(http.request.method) BY source.ip
| SORT hits DESC
```

## Post-hunt pivot — was a backdoor admin account created?

A successful exploitation creates an admin user. Hunt for new admin user creations in TeamCity audit logs:

```spl
index=teamcity earliest=-90d
| where action="user_created" AND user_role="SYSTEM_ADMIN"
| stats count, values(created_by) as creators by user_name
| sort -count
```

Any admin user created by `system` (rather than a known human admin) is a strong indicator of CVE-2024-27198 exploitation. Cross-reference creation time against the URL hunt to confirm.
