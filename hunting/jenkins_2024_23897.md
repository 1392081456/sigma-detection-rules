# Hunt — Jenkins CLI `@filename` Arbitrary File Read (CVE-2024-23897)

> Look for Jenkins CLI requests that include any argument prefixed with `@`. The Jenkins CLI's args4j parser expands `@filename` arguments by reading the named file from the server and inlining its content — exploitable for arbitrary file read pre-auth.

## Detection key

Jenkins exposes the CLI over both HTTP (`/cli/`) and TCP (default 50000). The args4j-based parser expands any argument starting with `@` as a file path. An attacker submits a CLI command with `@/etc/passwd` or `@/var/jenkins_home/secrets.key` to leak server-side files.

Common indicators:
- HTTP POST/GET to `/cli/` containing `@/` in the request body
- Connection to TCP port 50000 from non-trusted source
- Command `connect-node @/etc/passwd` and variants

## Splunk SPL

```spl
index=webserver earliest=-90d uri_path="*/cli*"
| eval body = if(isnotnull(http_body), http_body, "")
| where match(body, "@/[a-z]") OR match(uri_query, "@/[a-z]")
| stats count, values(http_method) as methods, values(uri_path) as paths by src_ip
| sort -count
```

## Microsoft Sentinel KQL

```kusto
let lookback = 90d;
W3CIISLog
| where TimeGenerated > ago(lookback)
| where csUriStem contains "/cli"
| where csUriQuery matches regex @"@/[a-z]"
| summarize hits = count(), methods = make_set(csMethod, 5), paths = make_set(csUriStem, 5) by cIP
| order by hits desc
```

## Elastic ES|QL

```esql
FROM logs-apache.access-*
| WHERE @timestamp > NOW() - 90 days
| WHERE url.path LIKE "*/cli*"
| WHERE url.query RLIKE ".*@/[a-z].*"
| STATS hits = COUNT(*), methods = VALUES(http.request.method) BY source.ip
| SORT hits DESC
```

## Post-hunt pivot — did the attacker pivot to RCE?

The arbitrary file read primitive lets an attacker leak Jenkins's `secrets.key` and then decrypt stored credentials. After leak, an attacker often logs in with a real admin credential. Hunt for new Jenkins admin logins from any IP that hit the CLI within the previous 24h:

```spl
index=jenkins source="*jenkins*audit*" earliest=-90d
| where action="login" AND user_role="administer"
| join type=left src_ip [
    search index=webserver uri_path="*/cli*" earliest=-90d
    | eval cli_time = _time
    | table src_ip cli_time
  ]
| where isnotnull(cli_time) AND _time > cli_time AND _time - cli_time < 86400
| table _time src_ip user cli_time
```
